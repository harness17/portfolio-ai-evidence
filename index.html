Interview talk track

面接での3分説明

採用担当者・技術担当者それぞれに向けた参照用の説明です。

採用担当者の方へ

  1. AIには実装・レビュー・記事化を任せています。設計判断・守秘判断・公開可否の決定は自分で行っています。
  2. Qiita/Zennに13本の技術記事を公開しています。いずれもCodexとClaude Codeの3ラウンド相互レビューを経ており、重大指摘0件で公開しています。
  3. YouTomはデスクトップアプリとしてv1.19.0まで継続リリースし、GitHub累計115DLで実稼働中です。
  4. このページ内の「課題 → 判断 → AI委譲範囲 → 検証 → 成果」の5項目が、各プロジェクトで自分が担った役割を示す軸です。

技術担当者の方へ

  1. 主な技術スタックはASP.NET Core（Identity・MVC）、Electron/React、YouTube Data API、Chrome Extensions（Manifest V3）です。
  2. AI活用の仕組みとして、Claude Code ↔ Codex間にhandoff多層契約（目的/範囲/権限/完成/検証/失敗の6層）を置き、Surgical Changesルールでdiff範囲を制御しています。手順はスキルファイルに書き出して再利用しています。
  3. 検証はdotnet test・npm test・Playwrightによる実動確認・Codexクロスレビューで行っています。
  4. 直近の設計判断例：YouTomのYouTube Data APIクォータ削減にRSSを採用、OAuth権限を読み取り専用に限定。Chrome拡張はMutationObserver再発火をバッジ差分更新で制御。
