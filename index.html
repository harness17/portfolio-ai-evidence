How I work

開発の進め方

AIを使って開発を効率化する一方で、設計・守秘・公開可否の判断は自分で行います。Claude CodeとCodexの連携を仕組み化し、品質を落とさず手戻りを減らしています。

自分で決めること

  • アーキテクチャと設計判断
  • 守秘・個人情報・公開可否の判断
  • 完成条件と検証観点の定義

仕組み化していること

  • AI間のhandoffを多層契約（目的/範囲/権限/完成/検証/失敗）で定義
  • 変更差分を絞るSurgical Changesルール
  • レビューゲートと、手順のskill化による再利用

検証のしかた

  • dotnet test / npm test
  • Playwrightによる実動確認
  • Claude Code↔Codexのクロスレビュー
